# Internationalization (i18n) & Localization Architecture

This document describes the internationalization architecture, translation management, client-side hydration strategy, dynamic routing, and localized asset handling for the **Trendy Fortune** website.

## 🛠️ Safe Client-Side i18n Pipeline (RSC Preservation)

### 1. The Strategy: Client-Side Translation Provider
Standard Next.js internationalization (via routing frameworks) requires deeply nested folder structures such as `/app/[locale]/page.tsx` and complex routing configurations. To maintain root-level SEO metadata, avoid folder reorganization, and maximize runtime performance:
- We implemented a **Client-Side Language Context** (`LanguageProvider`) in [LanguageContext.tsx](file:///c:/Users/pragy/OneDrive/Desktop/TrendyFortune/src/context/LanguageContext.tsx).
- Static JSON translation dictionaries are imported directly and served client-side using `next-intl`'s `<NextIntlClientProvider>`.

### 2. Hydration Mismatch Resolution & Mounting Flow
Since translations are chosen and resolved in the client's `localStorage`, rendering localized texts immediately on the server would trigger a React Hydration Mismatch (where the server-rendered HTML differs from the client-rendered output).
To eliminate hydration shifts and preserve a premium user experience, we employ the following architectural flow:

```
[Server Prerender] ────> Renders English text statically (fallback defaults)
                               │
                               ▼
[Client Mount] ────────> 1.2s Skeletons display instantly; page content is temporarily hidden
                               │
                               ▼
[Deferred State] ──────> useEffect retrieves `trendy_locale` from localStorage asynchronously
                               │
                               ▼
[Aync Set State] ──────> State changes in `setTimeout` turn, next-intl dictionary swaps dynamically
                               │
                               ▼
[Full Page Paint] ─────> Page fades in elegantly, showing translation with zero layout shift
```

## 🔤 Typographical System & Font Failsafes

Different Indian scripts require significantly different vertical line-heights and glyph alignments. To prevent text overlaps and preserve visual excellence:
1. **Google Fonts Configuration**: In [layout.tsx](file:///c:/Users/pragy/OneDrive/Desktop/TrendyFortune/src/app/layout.tsx), we pre-downloaded and mapped regional subsets:
   - `Noto_Sans_Devanagari`
   - `Noto_Sans_Bengali`
   - `Noto_Sans_Tamil`
   - `Noto_Sans_Malayalam`
2. **Unified CSS Variable Pipeline**: In [globals.css](file:///c:/Users/pragy/OneDrive/Desktop/TrendyFortune/src/app/globals.css), fonts are declared dynamically. When a regional script is rendered, the corresponding Noto fallback ensures legible glyph sizes:
   ```css
   :root {
     --font-noto-sans-regional: 'Noto Sans Devanagari', 'Noto Sans Bengali', 'Noto Sans Tamil', 'Noto Sans Malayalam', sans-serif;
   }
   
   body {
     font-family: var(--font-inter), var(--font-noto-sans-regional), sans-serif;
   }
   ```

## 🚀 Adding a New Language

Adding a new language takes 4 simple, safe steps:

### Step 1: Create the Translation Dictionary
Create a new JSON file under `src/messages/` (e.g., `src/messages/te.json` for Telugu). Copy the key-value schema from `en.json` and translate all values.

### Step 2: Import & Register the Locale
Open [LanguageContext.tsx](file:///c:/Users/pragy/OneDrive/Desktop/TrendyFortune/src/context/LanguageContext.tsx):
1. Import your JSON:
   ```typescript
   import te from "@/messages/te.json";
   ```
2. Add the locale to the `Locale` union:
   ```typescript
   export type Locale = "en" | "hi" | "bn" | "ta" | "ml" | "te";
   ```
3. Register it in `MESSAGES_MAP`:
   ```typescript
   const MESSAGES_MAP = { en, hi, bn, ta, ml, te };
   ```

### Step 3: Register in the Language Switcher
Open [LanguageSwitcher.tsx](file:///c:/Users/pragy/OneDrive/Desktop/TrendyFortune/src/components/LanguageSwitcher.tsx) and add your language details to the `LANGUAGES` config array:
```typescript
{ code: "te", label: "తెలుగు", flag: "🇮🇳" }
```

### Step 4: Add Google Font (If Needed)
If the script is not covered by standard fallbacks, import the Google Font inside [layout.tsx](file:///c:/Users/pragy/OneDrive/Desktop/TrendyFortune/src/app/layout.tsx) and append its variable class to the main `body` wrapping tag.
